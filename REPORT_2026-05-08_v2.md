## 1. 작업 내역

### 1-1. 텍스트 작성 로직 보강

회사 정보 세분화, 검색엔진 반영을 통해 콘텐츠 퀄리티 강화

- 회사 소개 / 작성 가이드 분리
- SEO 가이드 확장
- 키워드 구조 확장 ("브랜드 시드 키워드"(거의 변하지 않음)와 "트렌드 키워드"(주 단위 변동)를 별개 파일로 분리)
- 트렌드 수집 (네이버 데이터랩 + YouTube Data API 로 최근 1주 vs 이전 3주의 키워드 수집 후 반영)
- 드래프트 워크플로우 (워드프레스 "초안"으로 저장하고 편집 링크를 출력. 검토 후 수동 발행)

### 1-2. 본문 생성 프롬프트 개편

AI에게 단순히 "글 써줘"가 아니라 **에디터처럼 단계적으로 일하도록** 지시.

- "국내 자체 배터리 제조공장을 운영하는 시니어 콘텐츠 에디터" 정체성 고정.
- **9단계 생성 절차** 명시: 주제 추출 → 제목 공식 선택 → Hook 패턴 → 구조 설계 → 시각 요소 배치 → FAQ 판단 → 제조사 관점 삽입 → 마무리 → 자가 검증.
- **시각 요소 6종 사용 규칙**: 굵게 / 형광펜 / 인용 박스(주의·팁·참고) / 비교 테이블 / 체크리스트 / 구분선. 평문 위주의 단조로운 글에서 가독성 높은 글로.
- **이미지 프롬프트도 같은 규칙**으로: 단일 컨셉, 스타일 8종 가이드, 텍스트·사람 얼굴·로고 금지.

→ 결과물 품질이 눈에 띄게 좋아진 것을 4월 23일 실사용으로 확인.

### 1-3. 본문 생성 로컬화
- 사용 모델: `Qwen3-32B-4bit` (Apple Silicon용 MLX 런타임, 메모리 18GB 차지).
